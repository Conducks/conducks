Token Flow Optimization:
========================

BEFORE (Markdown Heavy):
Agent Request → Tool Handler → Markdown Generator → Response
                                      ↓
                              # 🛠️ CONDUCKS Tools
                              ## Available Tools
                              - **Tool 1**: Description
                              ↓
                         150-200 tokens consumed

AFTER (Plain Text TOON):
Agent Request → Tool Handler → Plain Text Generator → Response
                                      ↓
                              CONDUCKS Tools

                              1. Tool Name - Description
                              ↓
                         20-40 tokens consumed

Core Module (Essential Components)
======================================
types.ts      → Job, Task, Storage interfaces + enums
storage.ts    → TOON file read/write, job loading/saving
logic.ts      → Job generation, task logic, domain detection
config.ts     → Workspace paths, environment config, constants

config.ts functions:
  - getWorkspacePaths() → Resolves paths for workspace/project
  - Environment variable handling
  - Path security validation
  - Storage root configuration

storage.ts key operations:
  - loadCONDUCKSWorkspace() → Load entire workspace
  - saveJobForWorkspace() → Save job with backup to done-to-do
  -scanWorkspaceForJobs() → Legacy function (minimally used)
  - TOON format serialization/deserialization

logic.ts key operations:
  - Generate job IDs (sequential numbering)
  - Task ID generation within jobs
  - Job status calculations
  - Complexity analysis
